0 Overview — what this is
System boundary

repo-harness is not an app — it is a meta-tool that makes AI-agent work resumable, reviewable, and repo-local. Its job: inspect a target repo, install a file-backed workflow contract, route public command skills, run guard hooks during agent sessions, and verify the generated surfaces stay consistent. This checkout is self-hosting — it runs its own harness on itself (hook_source: "repo" pins .ai/hooks/ as the live runtime).

It is

  • A repo inspector + contract installer
  • A command-skill router (Claude + Codex)
  • An event-driven guard-hook runtime
  • A consistency verifier for generated surfaces

It is not

  • A product runtime, server, or DB service
  • An agent gateway or MCP server
  • An installer of external host tools (Waza/gstack/gbrain)
  • A vendor of external skill bodies (mermaid)

4 Flow 3 — CLI install layers

Entry: src/cli/index.ts (commander). The local-repo-harness binary installs in four nested layers — the model that makes its 20 subcommands legible. Targets are a registry of AgentTarget implementations (claude / codex); managed hook entries are tagged REPO_HARNESS_MANAGED=1 for idempotent, user-hook-preserving install.

L1User / global runtimeinit · update · setup

Writes global adapters + host context into ~/.claude/* and ~/.codex/*.

L2Repo-vendored runtimebootstrap

Installs the harness package into .ai/harness/tools/… + a .ai/harness/bin/ shim.

L3Repo adoptionadopt  (+ reclaim-runtime)

Applies repo template into .ai/, tasks/, plans/, project adapters, and the .gitignore / VCS boundary.

L4Host adaptersinstall · uninstall

Writes the tagged hook entries into host config. Runtime selection (auto / global-path / project-vendored-bun) picks PATH vs a repo-vendored bun wrapper.

Utility & read-only commands round out the surface: status, doctor, migrate, security scan, run, sprint, brain, capability-context, docs, vcs, tools.

6 Where state lives — truth vs derived
Contract / config · authoritative

Tracked. The behavioral source of truth.

  • .ai/harness/policy.json
  • .ai/harness/workflow-contract.json
  • .ai/context/capabilities.json
  • .ai/context/context-map.json

Ignored. Written by hooks during sessions.

  • .ai/harness/runs · events.jsonl
  • .ai/harness/checks · failures
  • .ai/harness/handoff · planning
  • markers: active-plan · active-worktree · active-sprint
Memory · advisory

Overridden by current repo state + evidence.

  • docs/researches/
  • tasks/lessons.md
  • gbrain (external knowledge)
